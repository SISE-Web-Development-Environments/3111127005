About Me

My name is Ortal Parpara, I am 26 years old, originally from Kfar Saba.
I currently live in Beer Sheva, studying Software and Information System Engineering in BGU.
In my free time (when there is no Coronavirus around) I love travelling and baking
(see the pictures below). In Beer Sheva I also do Crossfit training, volunteer with new immigrant
children with my scholarship “Keren Moshe” and I am an area manager in the “She Codes” organization.
I expect from this course to give me the ability to create a website
and mainly understand the correct way to design the code for a big web project.
In this exercise the main struggle for me was to decide on a design and then implement it
using the CSS features.

Q&A

1. שלושה הבדלים עיקריים בין web 1, web 2, web 3:

Web 1 התמקד בצפייה במידע שמפורסם ברשת, כלומר כפי שניתן לקרוא עיתונים, לצרוך מידע כתוב או הרצאה מוקלטת, כך ניתן היה לצפות ברשת. לא הייתה קיימת תקשורת בין המשתמש לבין התוכן ברשת.
ב Web 2 החלה אינטראקציה בין המשתמש לבין התוכן. כלומר, הקורא יכול לפרסם מידע ברשת וכן לצרוך מידע שאחרים מפרסמים. כאן החלה גם הכניסה של הרשתות החברתיות והאפשרות של כל משתמש להעלות תוכן, ולא רק של החברות הגדולות.
Web 3 הוא הרעיון שכל המידע יתוייג בצורה סמנטית, כך שגם מחשב וגם אדם יוכלו לקרוא אותו. כלומר, להפוך את התוכן למידע באופן מיידי כך שניתן להשתמש בו ולפעול לפיו. דבר זה יאפשר פרסונליזציה של הרשת לכל משתמש, המחשב יוכל להבין את פרטי המידע ברשת ומנועי חיפוש יוכלו למצוא בצורה מותאמת אישית את התוכן המתאים. הרשת כיום עדיין לא שם, למרות שיש שימוש בבינה מלאכותית לביצוע פרסונליזציה של המידע.
לכן, ניתן להגיד שההבדלים המרכזיים הם ההתממשקות של האדם אל הרשת, אופן צריכת המידע וכן אופן השימוש של המחשב במידע שנצרך ברשת.


2. תרחיש שימוש – קביעת פגישה בין קבוצת חברים ב zoom במהלך משבר הקורונה:

ב web 1 – הרשת לא יכולה לעזור בקביעת הפגישה, התוכן המוצג ברשת לא מאפשר למשתמשים להבין מתי כל אחד פנוי ומהן השעות האפשריות לקביעת הפגישה. המשתמשים יכולים ליצור קשר באימייל ולתאם פגישה, בדומה למצב בו היו קובעים את הפגישה בשיחת טלפון.
ב web 2 – המשתמשים יפרסמו את לוחות השנה שלהם דרך אפליקציית שיתוף כלשהי (גוגל לדוגמא), כל משתמש יכול לסמן את הזמנים הנוחים לו ולפי הזמנים הנוחים החופפים ניתן לקבוע פגישה בזמן שמתאים לכולם.
ב web 3 – לוחות השנה של החברים מפורסמים ברשת, המחשב יכול לקרוא את לוחות השנה מאחר וכל המידע מתוייג בצורה סמנטית. במצב זה, הפגישה נקבעת בצורה אוטומטית מבלי שחברי הקבוצה צריכים לבדוק את לוחות השנה אחד של השני. הפגישה מופיעה באופן אוטומטי ביומן של כל אחד מהחברים.
